# MMR — единый игровой рейтинг (foundation)

MMR (Match Making Rating в игровой терминологии, для игрока — просто «рейтинг»)
это **единый показатель общего прогресса** игрока в экосистеме Возни. Это
самостоятельная система: она копит очки за игровые действия и переводит их в
**ранги**. MMR — не валюта: его нельзя купить, передать, обменять или
потратить.

## Главные принципы

- **Изоляция.** MMR не связан с ешками (`users.balance` / `transactions`),
  репутацией (`reputation_entries`), счётчиком сообщений
  (`users.messages_count`), магазином, инвентарём, подарками, Combot и OIDC.
- **Сообщения не дают MMR.** Рейтинг растёт только за игровые действия.
- **История — источник правды.** Текущий MMR = `SUM(amount)` по журналу
  `mmr_entries`. Значение всегда пересчитывается из истории, отдельной
  «накопительной» колонки в `users` нет.
- **Только «MMR» в интерфейсе.** Слово XP в текстах для игрока не используется.

## Источники MMR (настраиваются в `app/settings/mmr.py`)

| Действие | Константа | Значение по умолчанию |
|----------|-----------|----------------------|
| Найден клад | `MMR_TREASURE` | 50 |
| Победа в дуэли | `MMR_DUEL_WIN` (+ участие) | 100 (+25) |
| Участие в дуэли (обоим) | `MMR_DUEL_PARTICIPATION` | 25 |
| Успешная ферма | `MMR_FARM` | 10 |
| Ачивка обычная | `MMR_ACHIEVEMENT` | 100 |
| Ачивка редкая (секретная) | `MMR_ACHIEVEMENT_RARE` | 250 |
| Ачивка легендарная | `MMR_ACHIEVEMENT_LEGENDARY` | 1000 |
| Ивент / админ-награда | произвольно | через `award_mmr(..., source=event/admin)` |

«Редкость» ачивки выводится из каталога достижений
(`app/settings/achievements.py`): категория `legend` → легендарная;
`hidden=True` / категория `secret` → редкая; остальные → обычные. Отдельного
поля rarity нет. См. `mmr_for_achievement()`.

## Ранги

Пороги и названия задаются списком `RANKS` в `app/settings/mmr.py` и
**меняются без миграций** (это просто отображение, не данные в БД):

| MMR | Ранг |
|-----|------|
| 0 | 🥉 Новичок |
| 1 000 | 🥈 Бродяга |
| 2 500 | 🥇 Авторитет |
| 5 000 | 💎 Легенда |
| 10 000 | 👑 Элита |
| 25 000 | 🔥 Боженька Возни |

Функция `get_rank(mmr)` возвращает ранг для значения.

## Команды

- `ммр` / `mmr` / `рейтинг` — карточка рейтинга (MMR + ранг). В ответ на
  чужое сообщение показывает рейтинг автора.
- `топммр` / `topmmr` — топ сообщества по рейтингу.

MMR и ранг также добавлены в карточку `/профиль`.

### Единая точка начисления

Все модули начисляют рейтинг через `award_mmr(session, player_id=..., amount=...,
source=..., reason=...)`. Функция пишет одну строку в `mmr_entries` и **не делает
commit** — его выполняет middleware после успешной обработки апдейта (как и в
остальном коде). `amount == 0` игнорируется.

Хук-точки уже встроены:
- `app/features/treasure/service.py` → `claim_treasure` (клад);
- `app/features/duel/service.py` → `accept_challenge` (победа + участие);
- `app/features/farm/service.py` → `do_farm` (успешная ферма);
- `app/features/achievements/service.py` → `_grant` (любая открытая ачивка,
  даже без ешко-награды).

## Таблица `mmr_entries`

| Колонка | Тип | Назначение |
|---------|-----|-----------|
| `id` | BigInteger PK | идентификатор записи |
| `player_id` | BigInteger | кому начислили/списали |
| `amount` | Integer | величина (>0 начисление, <0 списание) |
| `source` | String(32) | `treasure`/`duel`/`farm`/`achievement`/`event`/`admin` |
| `reason` | String(128), null | уточнение (код ачивки, `win`/`participation`, текст ивента) |
| `created_at` | DateTime(tz) | время |

Индексы: `ix_mmr_player` (player_id), `ix_mmr_player_created`
(player_id, created_at).

## Права (RBAC)

Добавлены разрешения (синхронно в `app/core/permissions.py` и
`v0-voznya/lib/auth/admin-permissions.ts`):

- `mmr.view` — просмотр рейтинга/истории (роль `support` и выше);
- `mmr.add` — начислить MMR (роль `admin` и выше);
- `mmr.remove` — списать MMR (роль `admin` и выше).

Эти права предназначены для будущей админ-панели/Mini App: награды за ивенты и
ручные коррекции выполняются через тот же `award_mmr` (с положительным или
отрицательным `amount` и `source="event"`/`"admin"`).

## Точки расширения

- **Новые источники** — добавить вызов `award_mmr(...)` в нужном месте и (по
  желанию) новую константу в настройках. Новая `source` не требует миграции.
- **Сезоны / распады** — можно ввести позже, добавив фильтр по `created_at`
  в запросах репозитория; журнал уже хранит время каждого изменения.
- **Анти-абуз** — при необходимости лимиты считаются по журналу `mmr_entries`
  (как антиспам в репутации), без отдельной таблицы.
- **Ивент-награды и админ-коррекции** — уже поддержаны через `source`
  `event`/`admin` и права `mmr.add` / `mmr.remove`.

## Чего система НЕ делает (сознательно)

- Не трогает баланс ешек и экономику.
- Не начисляется за сообщения/активность.
- Не является валютой и не участвует в магазине/подарках.
