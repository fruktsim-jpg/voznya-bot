## 1. 🗑️ АРХИТЕКТУРА ХРАНЕНИЯ ПАР СООБЩЕНИЙ

### Структура данных:

```python
# app/services/deletion.py

class DeletionService:
    def __init__(self, bot, sessionmaker, scheduler):
        self.bot = bot
        self.sessionmaker = sessionmaker
        self.scheduler = scheduler
        
        # Кэш: {(user_id, chat_id): (user_command_id, bot_response_id)}
        self._last_info_messages: dict[tuple[int, int], tuple[int, int]] = {}
```

### Логика работы:

```
Ключ: (user_id, chat_id)
Значение: (message_id команды пользователя, message_id ответа бота)

Пример:
{
    (123456, -100789): (5001, 5002),  # Андрей в чате Возня
    (789012, -100789): (5010, 5011),  # Мария в чате Возня
}
```

### Алгоритм:

```
1. Пользователь отправляет команду (например, /профиль)
2. Бот получает message_id команды = 5001
3. Бот отправляет ответ, получает message_id ответа = 5002
4. Проверяем кэш по ключу (user_id, chat_id)
5. Если есть старая пара (4990, 4991):
   - Удаляем message 4990 (старая команда) ← НЕМЕДЛЕННО
   - Удаляем message 4991 (старый ответ) ← НЕМЕДЛЕННО
6. Сохраняем новую пару (5001, 5002)
7. НЕ планируем автоудаление по таймеру
8. Новая пара живёт до следующего информационного окна
```

### ⚠️ ВАЖНО: Удаление только по замещению

**НЕ удалять по таймеру!**

Информационное окно живёт до тех пор, пока пользователь не откроет новое.

Пример:
```
10:00 - профиль открыт
10:05 - профиль всё ещё виден
10:10 - профиль всё ещё виден
10:15 - пользователь открывает баланс
      → профиль удаляется
      → команда "профиль" удаляется
      → баланс появляется
```

### Преимущества:

- ✅ Чистый чат (нет висящих команд)
- ✅ Одно активное окно на пользователя
- ✅ Простая реализация (кэш в памяти)
- ✅ Не требует БД

### ⚠️ ВАЖНО: Поддержка inline-кнопок

Система должна работать одинаково для:
- Команд (`/профиль`)
- Алиасов (`проф`)
- Inline-кнопок (`💰 Баланс`)

**Пример:**
```
Пользователь: профиль
Бот: [профиль с кнопкой "💰 Баланс"]

Пользователь нажимает кнопку "💰 Баланс"
→ Удаляется профиль
→ Удаляется команда "профиль"
→ Появляется баланс
```

**Реализация:**
- Callback-кнопки должны вызывать `schedule_info_message`
- Передавать `user_command_id` = ID сообщения с кнопкой
- Система автоматически удалит предыдущее окно

### Компромисс:

- ⚠️ После рестарта бота кэш теряется
- ✅ Старые окна останутся до следующего информационного запроса
