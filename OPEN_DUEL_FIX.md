# Исправление открытых вызовов на дуэль

## Проблема
Открытые вызовы на дуэль (команда `/бой 50` без указания цели) не работали из-за того, что поле `target_id` в таблице `pending_actions` было обязательным (NOT NULL).

## Решение
Внесены следующие изменения:

### 1. Модель данных (`app/models/pending_action.py`)
- Изменено поле `target_id` на nullable: `Mapped[int | None]`
- Добавлен комментарий о поддержке открытых вызовов

### 2. Обработчик дуэлей (`app/features/duel/handlers.py`)
- Добавлена логика обработки открытых вызовов в `cmd_duel()`
- Если цель не указана, парсится как открытый вызов
- Используется текст `DUEL_OPEN_CHALLENGE` вместо `DUEL_CHALLENGE`

### 3. Сервис дуэлей (`app/features/duel/service.py`)
- `create_challenge()` теперь принимает `target_id: int | None`
- `accept_challenge()` обновлён для обработки открытых вызовов:
  - Для открытых вызовов (target_id=NULL) проверяется, что принимающий не инициатор
  - Для персональных вызовов проверяется, что принимающий — именно цель

### 4. Миграция БД (`migrations/versions/0005_open_duels.py`)
- Создана миграция для изменения `pending_actions.target_id` на nullable

### 5. Тексты (`app/settings/texts.py`)
- Текст `DUEL_OPEN_CHALLENGE` уже был в коде (строка 221-224)

## Как использовать

### Открытый вызов (любой может принять):
```
/бой 50
```
Создаёт вызов на 50 ешек, который может принять любой участник чата (кроме инициатора).

### Персональный вызов (конкретному игроку):
```
/бой @username 50
```
или ответом (reply) на сообщение:
```
/бой 50
```

## Применение изменений

### На сервере с Docker:
```bash
cd voznya-bot
git pull
docker compose down
docker compose up -d --build
```

Миграция применится автоматически при запуске контейнера.

### Локально без Docker:
```bash
git pull
alembic upgrade head
python -m app.main
```

## Проверка работы

1. Создайте открытый вызов: `/бой 10`
2. Другой пользователь должен увидеть сообщение: "⚔️ [Имя] бросает вызов! Ставка 10 ешек"
3. Любой может принять кнопкой или командой `/го`
4. Инициатор не может принять свой же вызов

## Технические детали

- Открытые вызовы имеют `target_id=NULL` в БД
- Кнопка "Принять бой" работает для обоих типов вызовов
- Команда `/го` работает только для персональных вызовов (где указан target_id)
- Защита от race conditions сохранена (блокировки строк БД)
- Все проверки баланса работают корректно
