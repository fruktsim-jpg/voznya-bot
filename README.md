# 🤼 Бот «Возня»

Telegram-бот для чата **Возня**: внутренняя валюта «ешки», ферма, казино,
дуэли, клады, номинации «Пидор дня» и «Пара дня», браки, рейтинги и многое
другое. Проект сделан модульным, чтобы в будущем легко добавлять магазин,
достижения, лотереи и прочие механики.

> Эта инструкция написана так, чтобы её мог выполнить человек, который
> **не умеет программировать**. Просто выполняй шаги по порядку.

---

## 📋 Что умеет бот

> Все команды работают **со слэшем и без него** (`ферма` = `/ферма`) и имеют
> английские алиасы (`farm`, `balance`, ...).

| Команда (рус / англ) | Что делает |
|---|---|
| `ферма` / `farm` | Получить ешки (раз в 4 часа), серия за ежедневную активность |
| `баланс` / `balance` | Показать баланс ешек |
| `профиль` / `profile` | Карточка игрока: титул, место в рейтинге, статистика, прогресс |
| `ачивки` / `achievements` | Список достижений (открытые и закрытые) |
| `топ` / `top` | Рейтинг богачей |
| `топнеделя` / `weekly` | Лидеры по заработку за последние 7 дней |
| `казино сумма` / `casino` | Сыграть в казино (раз в час), есть кнопка «Повторить ставку» |
| `бой @ник ставка` / `duel` + `го` / `accept` | Дуэль на ешки (с кнопкой «Принять бой») |
| `снять` / `claim` | Забрать клад Возни (появляется 2–4 раза в сутки) |
| `пидор` / `pidor` | Пидор дня + топ за всё время |
| `пара` / `couple` | Пара дня |
| `жениться @ник` / `marry` + `да` / `yes` | Свадьба (с кнопкой «Согласиться») |
| `брак` / `marriage` | Информация о браке |
| `развод` / `divorce` + `подтвердить` / `confirm` | Развод |
| `семьи` / `families` | Рейтинг самых долгих браков |
| `помощь` / `help` | Список команд |

Команды для администраторов (см. `ADMIN_IDS`): `/выдать`, `/забрать`,
`/инфо`, `/клад`.

### 🏷 Титулы

Ранг игрока зависит от баланса (таблица легко расширяется в
`app/settings/titles.py`):

🌱 Щавель → 🍑 Персик → 🐀 Гой → 🍺 Бурмалда → 💊 Аптекарь → 🎰 Лудоман →
⚔️ Возняк → 🏆 Авторитет Возни → 👑 Король Возни → ☢️ Легенда Возни.

В профиле показывается текущий титул и прогресс до следующего.

### 🏅 Достижения

Система достижений (каталог — в `app/settings/achievements.py`) с бонусами за
открытие: первая ешка, фермер/барон, лудоман/казиношный дед, дуэлянт/гладиатор,
тысячник/магнат, кладоискатель/охотник за кладом, «Любовь существует» и
«Легенда Возни» (за получение всех достижений). Команда — `ачивки`.

### 🔘 Кнопки

- Под дуэлью — «⚔️ Принять бой», под предложением — «💍 Согласиться»
  (с защитой от чужих нажатий, повторов и просрочки).
- После казино — «🎰 Повторить ставку».
- После частых действий — быстрые кнопки «💊 Ферма / 💰 Баланс / 👤 Профиль / 🏆 Топ».

## 1️⃣ Создание и настройка бота в Telegram

1. Откройте [@BotFather](https://t.me/BotFather) → команда `/newbot` → задайте
   имя и юзернейм. Скопируйте **токен** (длинная строка вида `12345:AAabc...`).
2. **Отключите режим приватности** (важно! без этого бот не увидит активность
   участников для номинаций):
   - `/mybots` → выберите бота → **Bot Settings** → **Group Privacy** →
     **Turn off**.
3. Добавьте бота в чат «Возня» и сделайте его **администратором** с правом
   **«Удаление сообщений»** (иначе бот не сможет убирать спам и приветствия).

### Как узнать CHAT_ID и ADMIN_IDS

- **Ваш ID** (для `ADMIN_IDS`): напишите [@userinfobot](https://t.me/userinfobot)
  — он пришлёт ваш числовой ID.
- **ID чата** (для `CHAT_ID`): добавьте в чат [@getidsbot](https://t.me/getidsbot)
  или посмотрите ID в логах бота после запуска. У групп ID обычно начинается с
  `-100...`.

## 2️⃣ Установка на сервере (через Docker — рекомендуется)

Docker позволяет запустить бота и базу данных одной командой, ничего не
настраивая вручную.

```bash
# 1. Установить Docker (если ещё не установлен)
curl -fsSL https://get.docker.com | sh

# 2. Скачать проект
git clone <адрес-репозитория> voznya-bot
cd voznya-bot

# 3. Создать файл настроек из примера
cp .env.example .env
nano .env   # впишите BOT_TOKEN, CHAT_ID, ADMIN_IDS, придумайте POSTGRES_PASSWORD
```

В файле `.env` обязательно заполните:

- `BOT_TOKEN` — токен от BotFather;
- `CHAT_ID` — ID чата Возни;
- `ADMIN_IDS` — ваши ID через запятую (например `111111111,222222222`);
- `POSTGRES_PASSWORD` — придумайте любой пароль;
- `DATABASE_URL` — поправьте пароль в строке, чтобы он совпадал с `POSTGRES_PASSWORD`.

Запуск:

```bash
docker compose up -d
```

Готово! Бот:

- автоматически применит миграции базы данных при старте;
- будет работать 24/7 и перезапускаться сам при перезагрузке сервера;
- хранит все данные в томе PostgreSQL — **рестарт не теряет данные**.

Посмотреть логи:

```bash
docker compose logs -f bot
```

Остановить / запустить заново:

```bash
docker compose down      # остановить
docker compose up -d     # запустить
```

## 4️⃣ Резервные копии базы данных

Сделать бэкап вручную:

```bash
bash scripts/backup.sh
```

Дампы складываются в папку `backups/` (хранятся последние 30 штук).

Автоматический бэкап каждый день в 4:00 — добавьте строку в `crontab -e`:

```
0 4 * * * cd /полный/путь/к/voznya-bot && bash scripts/backup.sh >> backups/backup.log 2>&1
```

## ⚙️ Как менять настройки игры (без программирования)

Все числа и тексты вынесены в два понятных файла:

- **`app/settings/balance.py`** — кулдауны, шансы, награды, лимиты ставок.
  Меняйте только числа справа от `=`.
- **`app/settings/texts.py`** — все тексты сообщений. Меняйте текст в кавычках,
  не трогая части в фигурных скобках вроде `{amount}`.

После изменений перезапустите бота:

```bash
docker compose up -d --build
```

## 🧱 Архитектура проекта (для разработчиков)

Модульный монолит на **Python 3.12 + aiogram 3 + PostgreSQL + SQLAlchemy**.

```
app/
├── main.py            # точка входа: сборка бота, middleware, роутеров, планировщика
├── config.py          # настройки из .env
├── settings/          # баланс игры (числа) и тексты — правятся без кода
├── core/              # БД, логирование, планировщик, утилиты, фильтры
├── middlewares/       # сессия БД, фильтр чата, трекинг активности, антифлуд
├── models/            # таблицы БД (SQLAlchemy)
├── repositories/      # доступ к данным (запросы)
├── services/          # общая логика: экономика (ешки), кулдауны, удаления
└── features/          # МОДУЛИ-МЕХАНИКИ — каждая в своей папке
    ├── welcome/  farm/  casino/  duel/  treasure/
    ├── pidor/  para/  marriage/  profile/  balance/
    ├── ratings/  help/  admin/  achievements/  quick/
    └── ...
app/settings/titles.py        # таблица титулов (рангов)
app/settings/achievements.py  # каталог достижений
app/core/money.py             # склонение и форматирование ешек
migrations/            # миграции БД (Alembic)
```

Ключевые принципы:

- **Вся валюта проходит через `services/economy.py`** (единый кошелёк + журнал
  транзакций). Это основа для будущей экономики (банк, магазин, лотереи).
- **Атомарность и защита от гонок**: критичные операции (клад, выбор номинаций,
  дуэли) используют блокировки строк и уникальные ограничения БД.
- **Расширяемость**: новая механика = новая папка в `features/` со своим
  `router`, подключаемая в `app/features/__init__.py`. Кулдауны и номинации
  устроены универсально и переиспользуются.

### Технологии

- **aiogram 3** — асинхронный фреймворк для Telegram-ботов;
- **PostgreSQL** — надёжная БД с транзакциями (нужны против гонок);
- **SQLAlchemy 2.0 (async) + Alembic** — ORM и миграции;
- **APScheduler** — расписание (клады, отложенные удаления, суточные сбросы);
- **Docker + docker-compose** — простой запуск и обновление.

## 🔁 Запуск без Docker (опционально)

Если хотите запускать вручную (нужен установленный PostgreSQL):

```bash
pip install -r requirements.txt
cp .env.example .env   # заполнить, DATABASE_URL указать на свой PostgreSQL
alembic upgrade head   # применить миграции
python -m app.main     # запустить бота
```
