# COMBOT MIGRATION — исследование API

Исследование Combot API v5 на предмет импорта исторической статистики чата
**ВОЗНЯ** в собственную базу бота. Только разведка: код импорта не писался,
миграции не создавались.

- Дата: 2026-06-06
- Чат: `-1003278808005`
- Базовый URL: `https://api.combot.org/v5/`
- Авторизация: query-параметр `api_key=<key>` (ключ не коммитим в репозиторий)
- Формат ответа: `{"ok":true,"result":{...}}`; ошибка —
  `{"ok":false,"error_code":404,"description":"Not Found"}`
- Время: все таймстампы в **миллисекундах** Unix (например `1780178400000`).

> ⚠️ API-ключ из задачи — боевой. В этом документе он намеренно не приводится.
> Хранить как секрет (env / Vault), не пушить в git.

## 1. Какие endpoints реально существуют

Перебрал ~30 кандидатов. Живых оказалось **два**, оба с префиксом `channel_`.
Всё остальное (`users`, `members`, `messages`, `activity`, `leaderboard`,
`top_users`, `stats`, `analytics`, `overview`, `channel_messages`,
`channel_leaderboard`, `channel_top`, `channel_activity` и т.д.) → `404 Not Found`.

| Endpoint | Статус | Что отдаёт |
|----------|--------|------------|
| `channel_analytics` | ✅ 200 | Агрегированные тайм-серии (сообщения, активные, joins/leaves) + последние вступившие/вышедшие |
| `channel_users` | ✅ 200 | Полный список участников с накопленной статистикой (сообщения, XP, репутация, дата входа). Пагинация. |
| всё прочее | ❌ 404 | Не существует |

Вывод: отдельных endpoint'ов «messages», «leaderboard», «activity» нет —
их данные «зашиты» внутрь двух работающих ручек.

## 2. `channel_analytics` — агрегированные тайм-серии

### URL и параметры
```
GET https://api.combot.org/v5/channel_analytics
  ?chat_id=-1003278808005
  &from=<ms>          # начало периода (Unix ms)
  &to=<ms>            # конец периода (Unix ms)
  &api_key=<key>
```
Гранулярность выбирается автоматически по ширине окна (`day` для недельного
диапазона). Endpoint всегда возвращает ещё и **предыдущий** период такой же
длины (`*_prev`) для сравнения.

### Структура ответа (ключевое)
```jsonc
{
  "ok": true,
  "result": {
    "joined":        [[ts, n], ...],   // вступления по дням
    "joined_prev":   [[ts, n], ...],
    "joined_total": 21,
    "left":          [[ts, n], ...],   // выходы по дням
    "left_total": 13,
    "messages":      [[ts, n], ...],   // сообщения по дням
    "messages_total": 5256,
    "active_users":  [[ts, n], ...],   // активные пользователи по дням
    "active_users_total": 110,
    "hours": [[hour, weekday, count], ...],  // тепловая карта 24×7
    "is_channel": false,
    "last_joined_users": [ { "user_id", "u":[{title,username}], "joined", "avatar"(base64 jpeg) }, ... ],
    "last_left_users":   [ { "user_id", "u":[...], "left", "avatar" }, ... ],
    "comparison": { "granularity":"day", "current_start", "current_end", "prev_start", "prev_end" }
  }
}
```

Заметки по полям:
- `messages` / `active_users` / `joined` / `left` — массивы пар `[timestamp_ms, value]`.
- `hours` — тройки `[час 0–23, день_недели 0–6, кол-во]`: готовая heat-map
  «когда чат живой».
- `last_joined_users` / `last_left_users` — до ~10 последних, **с аватаркой в
  base64** (jpeg). Тяжёлые (ответ за неделю ≈ 53 КБ в основном из-за аватаров).
- `avatar` есть не у всех (приватность / нет фото).

### Полезность для Возни
- Графики «сообщений в день», «активных в день», «приток/отток» на сайте /live
  и в админке.
- Heat-map активности (`hours`) — украшение дашборда, плюс подсказка по времени
  ивентов.
- НО: это **агрегаты**, без привязки к конкретному user_id (кроме last_*).
  Для персональной истории игроков нужен `channel_users` (ниже).

## 3. `channel_users` — пер-юзерная статистика (главная находка)

### URL и параметры
```
GET https://api.combot.org/v5/channel_users
  ?chat_id=-1003278808005
  &from=<ms>&to=<ms>
  &api_key=<key>
  &page=<n>           # 0-based, по умолчанию 0
  &limit=<n>          # размер страницы; limit=500 вернул все 405 за раз
```
Пагинация — через `page` + `limit` (параметр `offset` игнорируется). В ответе:
```jsonc
"pagination": { "page":0, "limit":25, "has_next":true, "has_prev":false, "total":405, "total_pages":17 }
```
`limit=500&page=0` отдал все 405 участников одним запросом (`total_pages:1`),
так что весь ростер забирается за 1 вызов.

### Структура записи пользователя
```jsonc
{
  "user_id": 8354006262,
  "u": [ { "title":"Имя", "username":"nick", "language_code":"ru" } ],
  "joined": 1778870503000,   // дата входа (ms); может быть пустой у старых
  "dsj":    0,               // days since joined (дней в чате); пустой, если joined неизвестен
  "messages": 14037,         // всего сообщений за период
  "last_message": 1780703783000,
  "rep": 20,                 // репутация (Combot reputation)
  "xp":  19232               // опыт/уровни Combot
}
```
`result.features`: `{ "levels_enabled":true, "reputation_enabled":true, "warns_enabled":false }`
— то есть XP/уровни и репутация включены, варны выключены (поля warns не будет).

### Полезность для Возни — максимальная
Это фактически **готовая таблица игроков с накопленной активностью**:
- `messages` → исторический счётчик сообщений (у нас `users.messages_count`).
- `joined` → «старожилы»: сортировка по дате входа.
- `xp` / `rep` → можно конвертировать в стартовый баланс/репутацию или ачивки.
- `username` / `title` → сопоставление с `users` по `user_id`.
- `last_message` → активность/«живость» для антиспама и наград.

## 4. Полнота исторических данных

Проверил широкий диапазон `from=1577836800000` (1 янв 2020) → `to` сегодня:

| Метрика | Значение за весь период |
|---------|-------------------------|
| `messages_total` | **94 960** |
| `joined_total` | 571 |
| тайм-серия `messages` | 2348 дневных бакетов (≈ с момента появления чата в Combot) |
| `channel_users.total` | 405 участников (все, кто писал/состоял) |

Вывод: Combot отдаёт историю **с самого начала наблюдения** (не только за
последнюю неделю/месяц). Можно выгрузить всё разом.

Ограничение: история ограничена моментом, когда Combot **добавили в чат**.
Сообщения до подключения Combot в их базе отсутствуют — это не «вся история
Telegram», а «вся история с момента подключения бота Combot».

## 5. Сводка по вопросам задачи

| Вопрос | Ответ | Источник |
|--------|-------|----------|
| Статистика по пользователям? | ✅ Да, по каждому | `channel_users` |

## 6. Что можно импортировать в Возню

| Данные | Откуда | Куда лечь у нас | Замечания |
|--------|--------|-----------------|-----------|
| **Старожилы** | `channel_users.joined` (asc) | флаг/ачивка «старожил», ранний бонус | у части старых юзеров `joined` пустой → fallback на «известен Combot» |
| **Топ активности** | `channel_users.messages` (desc) | стартовый баланс/титулы, лидерборд | top-1 ≈ 14 037 сообщений |
| **Исторические достижения** | `xp`, `rep`, пороги `messages` | ачивки/бейджи при первом входе | XP/rep включены (`features`) |
| **Статистика сообщений** | `channel_users.messages`, `last_message` | `users.messages_count`, «последняя активность» | суммарно по чату 94 960 |
| **Ежедневная активность** | `analytics.messages/active_users/joined/left` | таблица дневных метрик для графиков /live | пары `[ts_ms, value]` |
| **Heat-map активности** | `analytics.hours` | виджет «когда чат живой» | тройки `[hour, weekday, count]` |
| **Аватары** (опц.) | `last_*_users[].avatar` (base64) | кэш аватарок | тяжёлые, выборочно |

Стратегия импорта (на будущее, без реализации сейчас):
1. Один разовый вызов `channel_users?limit=500` → полный ростер 405 чел.
2. Матчинг по `user_id` с нашей `users`; для отсутствующих — создать «теневые»
   записи или импортировать при первом появлении.
3. `channel_analytics` за весь период → отдельная таблица дневных метрик для
   графиков (не привязана к игрокам).
4. XP/rep/messages → конвертация в наши единицы по согласованным коэффициентам
   (правила экономики — отдельное решение, не часть этого аудита).

## 7. Оценка: держать ли Combot и можно ли уйти

### Стоит ли держать подписку
- **Краткосрочно — да**, пока не сделан импорт и пока бот Возни сам не накопил
  достаточно собственной статистики. Combot — единственный источник
  **ретроспективы** (94 960 сообщений, 405 участников, история по дням). Если
  подписку отключить и потерять доступ к API до выгрузки — эти данные не
  восстановить.
- Combot также продолжает считать XP/rep/joins «бесплатно» для нас, пока свой
  трекинг не дозрел.

### Можно ли полностью забрать историю и потом отказаться
- **Да, технически реально.** Оба нужных endpoint'а отдают всё:
  - `channel_users?limit=500` — полный пер-юзерный срез одним запросом;
  - `channel_analytics?from=<начало>&to=<сейчас>` — вся дневная история.
- После одноразовой выгрузки в нашу БД зависимость от Combot для **истории**
  исчезает.
- Условие ухода: наш бот должен сам считать то, что сейчас считает Combot
  (сообщения, joins/leaves, активность, при желании XP/rep). Эта телеметрия
  у нас частично уже есть (`users.messages_count`), часть (joins/leaves,
  дневные агрегаты, heat-map) надо начать писать самим.

### Рекомендация
1. **Сейчас:** сделать разовый дамп `channel_users` (limit=500) и
   `channel_analytics` за весь период в «сырые» таблицы/файлы — зафиксировать
   историю, пока есть доступ. (Это следующий шаг; в рамках данной задачи —
   только исследование.)
2. **Параллельно:** включить собственный сбор дневных метрик и joins/leaves в
   боте, чтобы перестать зависеть от Combot для новых данных.
3. **Потом:** когда дамп сделан и свой трекинг работает ≥ 1–2 месяца и сходится
   с Combot — подписку можно не продлевать без потери данных.

Итог: Combot **не обязателен в долгую**, но прямо сейчас это безопасная «точка
сохранения» истории — отключать до выгрузки нельзя.
