# Release 2.2 — Алиасы на сайт, инвентарь, Telegram Login, редкие дропы

## P0 — Полный перевод алиасов кейсов/магазина на сайт

Аудит всех команд/коллбэков (см. также `RELEASE_2_2_SITE_FIRST_MAP.md`). После
этого раунда ВНУТРИ Telegram больше нет ни одного пути открытия кейса или
покупки подарка:

| Точка входа | Было | Стало |
|-------------|------|-------|
| `/кейсы`, `/cases` | список+открытие | карточка → `/cases` |
| `/кейс <код>` | дроп-лист + кнопка «Открыть» в чате | дроп-лист + кнопка на сайт |
| `/открыть`, `/open` | вскрытие в чате | карточка → `/cases` |
| callback `case:open:*` | вскрывал кейс (старые сообщения) | теперь ведёт на сайт |
| `/магазин`, `/shop`, `/подарки`, `/gifts` | витрина+покупка | карточка → `/gifts` |
| callback `gift:buy:*` | списывал ешки (старые сообщения) | теперь ведёт на сайт |

Серверный конвейер (`open_case`, `buy_gift`, `deliver_gift`) сохранён — его
использует сайт через внутренний API и админ. Кнопки `gift:keep/sell/withdraw`
(управление уже выпавшим призом) остаются: это не открытие/покупка, а действия
над предметом.

## P0 — Telegram Login: корневая причина

**Симптомы:** после входа аккаунт «не сохраняется», игрок считается
неавторизованным, переход в Telegram/обратно ломается.

**Root cause (не костыль):** оба auth-callback (`/api/auth/telegram` и OIDC
`/api/auth/telegram/oidc/callback`) строили редирект через
`new URL(path, request.url)`. За реверс-прокси (Vercel/nginx/Cloudflare)
`request.url` несёт ВНУТРЕННИЙ origin — часто `http://` и внутренний хост.
Браузер после успешного входа уходил на этот `http://`-URL, а только что
выставленная `Secure`-кука сессии на `http`-навигацию НЕ отправляется → профиль
открывается «неавторизованным». Сессия по сути терялась сразу после логина.

**Фикс:** `lib/auth/external-origin.ts` — `externalOrigin(request)` строит
ВНЕШНИЙ origin по приоритету: `PUBLIC_SITE_URL`/`NEXT_PUBLIC_SITE_URL` →
`x-forwarded-proto` + `x-forwarded-host` → `host` (в prod — https) → fallback.
Все редиректы обоих callback переведены на `externalUrl(request, path)`. Теперь
браузер остаётся на реальном `https://`-origin, кука уходит, сессия держится.

Сама проверка подписи (Login Widget HMAC, WebApp initData, OIDC) и форма
JWT/cookie не менялись — проблема была только в origin редиректа.

Рекомендация по окружению: задать `PUBLIC_SITE_URL=https://voznya.nl` (самый
надёжный путь, не зависит от заголовков прокси). Mini App (`verifyWebAppInitData`)
уже реализован в `lib/auth/telegram.ts` и переиспользует ту же сессию.

## P0 — Глобальные уведомления о редких дропах

`app/features/cases/rare_drop.py` — событие сообщества при по-настоящему редком
дропе. Анонс в общий чат (`settings.chat_id`), формат:

```
🎉 КРУПНАЯ НАХОДКА
Игрок: @username
Кейс: <название>
Предмет: 💍 Кольцо
Стоимость: 1450 ешек
Шанс: 0.05%
```

Отдельные акценты: `⭐ PREMIUM ВЫПАЛ!`, `💎 ДЖЕКПОТ!`, `🎁 РЕДКИЙ ПОДАРОК!`.

Анти-спам: анонс ТОЛЬКО при выполнении хотя бы одного порога (конфиг
`app/config.py`, любой можно отключить):
- `rare_drop_announce_jackpot` — любой джекпот;
- `rare_drop_announce_gift` — любой Telegram Gift / Premium;
- `rare_drop_chance_pct` (1.0%) — шанс ниже порога;
- `rare_drop_min_value` (1000) — стоимость не ниже порога;
- общий рубильник `rare_drop_announce_enabled`.

Точка вызова — внутренний API открытия (`/internal/cases/open`), т.к. кейсы
теперь открываются только на сайте. Best-effort: ошибка анонса не влияет на
открытие. Для шанса в `OpenResult` добавлено поле `reward_chance_pct`
(вес/сумма весов × 100).

## P0 — Социальное доказательство (следующий шаг)

Те же редкие события должны жить в live feed / на главной / в кейсах / в
профиле. Backend-данные для этого уже есть: открытия пишутся в `case_openings`
(включая `roll`, `weight_snapshot`, `reward_*`), а редкость дропа вычислима из
веса. План фронта (отдельная задача):
- лента `lib/live` уже читает события — добавить выборку «редких» по тем же
  порогам (джекпот/gift/премиум/низкий шанс/высокая стоимость);
- карточка «крупная находка» на главной и на странице кейсов;
- бейдж редкого дропа в профиле игрока.

Это идёт после визуальной полировки главной и статистики кейсов.
