### Реализованная врезка (что уже зашито в код)

Сезонный MMR начисляется ВМЕСТЕ с lifetime MMR — врезка сделана в самом
`mmr.service.award_mmr` (`_mirror_to_season`): один вызов покрывает ВСЕ источники
(ферма/клад/дуэль/ачивки/event), потому что все они идут через `award_mmr`. При
отсутствии активного сезона зеркалирование — no-op.

* Daily/Streak: команды `бонус`/`daily`/`дейли` → `claim_daily` (touch_streak +
  анти-двойной клейм через uniq + начисление ешек reason='daily').
* Weekly missions: `progress_mission` врезан в `farm/service` (метрика farm) и
  `duel/service` (метрика duel_win при победе); выдача награды при достижении
  target идемпотентна по `claimed_at`. Команда `миссии` показывает прогресс.
* Анти-дуэль-фарм: `duel/service._participation_mmr` отдаёт MMR за участие только
  ≤ K раз/день и только с РАЗНЫМИ оппонентами (оппонент закодирован в
  `mmr_entries.reason`). Победа (MMR_DUEL_WIN) засчитывается всегда.
* Сезон: команды `сезон`, `топсезон`; админские `стартсезон`/`финалсезон`.

## 4. Инструкции деплоя

```bash
# 1. Подтянуть код, проверить ветку.
git pull

# 2. Бэкап БД ДО любых изменений (обязательно — вайп необратим).
pg_dump "$DATABASE_URL" > backup_pre_season1_$(date +%F).sql

# 3. Применить схему сезона (НЕ вайп).
alembic upgrade 0033_season_system

# 4. (после готовности сид-миграции) засеять сезонный кейс/лимитки.
alembic upgrade 0035_seed_season_1

# 5. Перезапустить бота и сайт, прогнать тесты.
pytest -q
docker compose up -d --build
```

Деплой схемы безопасен и обратим (`downgrade` есть). Вайп выполняется отдельно
(шаг 6 ниже), только когда всё готово к старту.

## 5. Инструкции запуска сезона

1. Убедиться, что схема (0033) и сезонный контент (0035) применены.
2. Выполнить вайп (см. §6) в окно обслуживания.
3. Стартовать сезон: создать запись `seasons` (active=True, name="Сезон 1",
   `ends_at = now + 56 дней`). Программно — `repositories.season.start_season`
   (через будущую админ-кнопку `/admin/season` или разовый скрипт).
4. Объявить в чате: старт, дивизионы, награды, длительность.
5. Мониторинг первые 72 часа: `/admin/economy` (инфляция, аномалии перелива),
   `/admin/cases` (фактический RTP), `/admin/shop` (что покупают).

## 7. Список рисков

| Риск | Митигация |
|---|---|
| Вайп необратим, ошибка → потеря данных | Обязательный `pg_dump` ДО вайпа; вайп best-effort по таблицам (не падает на отсутствующих) |
| Сезонный MMR разойдётся с журналом | Проекция двигается атомарным `UPDATE ... season_mmr + :d`; есть журнал для пересчёта |
| Антиабуз-хуки не включены до старта | enforcement лимитов (перелив ешек, дуэль-фарм, порог фермы) — БЛОКЕР старта, см. чек-лист |
| Daily/weekly раздувают mint | Значения откалиброваны (≤ ~60 ешек/день суммарно); пороги в `settings.season` |
| Сезонные лимитки станут ешко-краном | `SEASON_LIMITED_SELL_RATE = 0.0` (продать обратно нельзя) |
| Двойной daily-claim при гонке | Uniq-констрейнт `(player_id, claim_date)` в `daily_claims` |
| Старые аккаунты имеют преимущество | Вайп обнуляет всё игровое; messages_count не даёт ни ешек, ни MMR |

### Чек-лист блокеров перед стартом (P0 enforcement)
- [x] Сезонный MMR начисляется в farm/treasure/duel/achievements (через `award_mmr`).
- [x] Анти-дуэль-фарм (`DUEL_MMR_PARTICIPATION_MAX_PER_DAY`, разные оппоненты).
- [x] Daily/missions хендлеры подключены (`season/handlers.py`).
- [x] Админ-управление сезоном (старт/финал) — команды `стартсезон`/`финалсезон`.
- [~] Лимит передачи ешек — значения готовы; активной команды передачи ешек в
  боте нет (вектор закрыт самим отсутствием механики). Включить в момент
  появления передачи.
- [~] Порог фермы (`FARM_MIN_MESSAGES`, `FARM_MIN_ACCOUNT_AGE_HOURS`) — значения
  готовы; врезка в фарм-гейт — P2 (сложный анти-мультиаккаунт отложен владельцем).

## Критерий успеха

Проект способен жить несколько сезонов без полного пересмотра экономики, потому
что: (1) вайп — плановый сезонный механизм, а не аварийный; (2) нет бесконечных
кранов (казино EV<1, кейсы RTP<1, sell<buy, daily/weekly закапы); (3) антиабуз
закрывает перелив ешек/MMR и мультиаккаунт-ферму; (4) сезонный сброс снимает
накопленный дисбаланс раз в 56 дней штатно.
