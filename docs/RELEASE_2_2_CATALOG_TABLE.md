# Release 2.2 — Полная таблица каталога подарков (источник истины)

Сводка собрана из миграций-сидов: 0018/0020 (база), 0024 (полный каталог +
Premium), 0028 (реальные telegram_gift_id), 0029 (лимитные сезонные), 0030
(фикс канона лимиток). Курс: **1★ = 10 ешек** (`ESHKI_PER_STAR`). Цена продажи
= `floor(price_eshki × 0.70)` (`ITEM_SELL_RATE = 0.70`).

Лимитный = `telegram_gift_id` входит в канон из 8 ID (см. `lib/limited-gifts.ts`
и миграцию 0030). Экономика лимитки: 50★, цена = обычная 50★-цена (700) × 1.5 =
**1050**, продажа = **735**.

## Сводка по консистентности (что было не так и что исправлено)

- **Лимитность вычислялась по-разному и неверно.** `lib/gifts.ts` (магазин)
  определял лимитку как `stock != null`, но у лимиток `stock=NULL` → они НИКОГДА
  не помечались лимитными в магазине. `lib/inventory-list.ts` наоборот считал
  ЛЮБОЙ pending-подарок лимитным. → Введён единый источник `lib/limited-gifts.ts`
  (`isLimitedGiftId` по telegram_gift_id), оба места переведены на него. В
  инвентаре добавлен бейдж «лимит».
- **9-я лимитка вне канона.** `gift_builder_bear` отсутствует в 8 ID источника
  истины → миграция 0030 деактивирует её.
- **Экономика лимиток** (50★ → 1050 ешек, продажа 735) подтверждена и
  идемпотентно зафиксирована в 0030.
