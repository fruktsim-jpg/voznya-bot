## 4. Экономика — проверка цен
Текущие `price_eshki` укладываются в сетку `star × 10 × наценка`:
- 15★ → 200 (×1.33), 25★ → 320 (×1.28), 50★ → 700 (×1.40), 100★ → 1450 (×1.45).
Наценка растёт с тиром (10–45%) — соответствует VOZNYA_ECONOMY_V2 §3. Менять
цены без обновления списка номиналов не требуется.

## 5. Влияние на кейсы (требует решения перед ре-сидом)
- Если 3.2 = переименование: коды в дроп-листах не меняются, правим только
  `gift_catalog` (имя/эмодзи) — кейсы трогать не нужно.
- Если выводим `gift_cup`/`gift_diamond` из ассортимента: нужен новый сид кейсов
  (0029) с пересчётом весов 100★-тира на их замену, иначе кейс выдаст приз,
  которого нет в магазине.
- 8 уникальных можно добавить в топ-тир `case_jackpot`/`case_collector` после
  назначения им цены и редкости.

## 6. Что сделано в этой итерации
- Миграция `0028_sync_gift_catalog_real_ids` — реальные ID для 9 базовых
  подарков (идемпотентно, обратимо).
- Миграция `0029_seed_collectible_gifts` — 9 сезонных лимитных подарков 50★
  с реальными ID (идемпотентно, обратимо).
- `lib/gifts-ux.ts` (сайт) — эмодзи для новых collectible (🎄/🧸/🤡 и т.д.),
  чтобы они различались в магазине и инвентаре.
- Эта ревизия (аудит + таблица экономики + открытые вопросы).
