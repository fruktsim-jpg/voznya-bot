# Возня — Мобильный UX/UI аудит

> Взгляд продуктового дизайнера, а не разработчика. Цель: довести сайт до состояния
> приложения для ежедневного использования с телефона. Никаких новых механик —
> только плотность, скорость, иерархия.

Аудит основан на реальном коде фронтенда (`v0-voznya`), а не на догадках. По каждому
пункту указан конкретный файл/блок.

---

## 0. Диагноз одним абзацем

Сайт построен как **набор маркетинговых лендингов**, обёрнутых в app-навигацию
(`bottom-nav.tsx` — это правильно, единственный по-настоящему «приложенческий»
элемент). Каждая страница начинается с большого `PageHero` (эмодзи + заголовок
`text-6xl` + абзац описания + пульсирующий бейдж), затем идут «гид новичку»,
«легенды», «социальные доказательства», поясняющие абзацы внизу. Контент тонет
в обёртке. На телефоне первый экран почти любой страницы — это **заголовок и
воздух**, а не сам контент (кейсы, подарки, статистика).

Главная причина всех 7 проблем из задания одна: **страницы объясняют сами себя
текстом, вместо того чтобы показывать контент сразу.** Приложение не объясняет —
оно показывает.

## 1. Полный аудит мобильного UX по страницам

Формат для каждой: что перегружено → что удалить → что свернуть → что видно сразу.

### 1.1 Главная (`app/page.tsx`)

10 секций подряд в одном `<main>`:
`Hero → LiveStats → CommunityActivity → TopMembers → Platforms → BotEcosystem →
About → FinalCta → SiteFooter → StickyCta`.

Это классический лендинг. Для приложения для ежедневного использования он
длинный и «продающий» — но человек уже внутри, его не надо продавать.

- **Перегружено:**
  - Дублирование «масштаба сообщества»: `Hero` уже тянет messages-счётчик,
    `LiveStats` показывает 6 крупных метрик (`live-stats.tsx`), а потом ещё
    `TopMembers`. Три блока про «нас много».
  - `Platforms` + `BotEcosystem` + `About` + `FinalCta` — четыре подряд
    маркетинговых блока «что такое Возня и где мы есть». Для своего это шум.
  - `StickyCta` поверх `bottom-nav` — две плавающих панели снизу конкурируют
    за один и тот же палец.
- **Удалить (для залогиненного / повторного визита):** `About`, `FinalCta`,
  `Platforms`/`BotEcosystem` свернуть в один компактный блок «Где мы есть».
  `StickyCta` убрать на мобиле — её роль уже выполняет bottom-nav.
- **Свернуть:** `LiveStats` 6 метрик → 3 самые сильные (Участники, Сообщений,
  Ешек в обороте) в одну плотную строку. Полную статистику и так открывает
  кнопка «Вся статистика и топы».
- **Видно сразу без скролла должно быть:** кто я (баланс/ник, если залогинен) +
  быстрые входы в Кейсы / Подарки / Live + 1 строка живой ленты. То есть
  **дашборд, а не презентация.** Сейчас первый экран — это `Hero` с лозунгом.

### 1.2 Кейсы (`app/cases/page.tsx` + `case-card.tsx`)

Самая болезненная страница и главный приоритет.

- **Перегружено:**
  - `PageHero` с абзацем на 4 строки про «трать ешки — лови Gifts…».
  - Блок «гид новичку» — сетка из 4 плиток (📦 N кейса / 🎁 Gifts / ⭐ Premium /
    💎 джекпот). Это ещё один экран до первого кейса.
  - Сама `CaseCard` — гигантская: капсула высотой `h-32` с эмодзи `text-6xl`,
    лента редкостей, заголовок, **описание**, строка «шанс редкой награды»,
    бейджи Gift/Premium, строка джекпота, заголовок «Лучшие награды» + **список
    из 4 наград** (`RewardRow` по `py-2` каждая), строка «и ещё N наград», и
    только потом кнопка «Открыть». Одна карточка — это **больше одного экрана
    телефона.**
  - Внизу ещё абзац про «шансы рассчитаны из весов дроп-листа».
- **Удалить с карточки:** описание кейса, ленту редкостей (`RarityStrip` —
  чистая декорация, решение не помогает), заголовок «Лучшие награды», список
  `RewardRow`, строку «и ещё N наград», строку «шанс редкой награды»
  (дублирует Gift/Premium шансы). Всё это — в детали.
- **Свернуть:** «гид новичку» — в одну строчку-легенду или вовсе убрать (иконки
  Gift/Premium на самих карточках уже объясняют суть). Hero ужать до строки.
- **Видно сразу:** 2–3 кейса целиком (см. раздел 2 — новый дизайн).

### 1.3 Подарки (`app/gifts/page.tsx` + `gift-card.tsx`)

- **Перегружено:**
  - `PageHero` + блок «✨ Трофеи коллекции» (3 крупные карточки) + «Легенда
    редкостей» (плашки всех редкостей) + «Вся коллекция». До самой коллекции —
    два экрана.
  - «Трофеи» дублируют «Всю коллекцию»: те же подарки показаны дважды (трофеи —
    это просто топ той же сортировки).
  - Карточка (`CollectibleTile`) сравнительно ок, но `subtitle` (описание
    подарка) почти всегда шум, а строка «осталось N · забрали N» — мелкий текст,
    который сливается.
- **Удалить:** блок «Трофеи» (дубль), «Легенду редкостей» (цвет рамки и так
  кодирует редкость — подпись не нужна на каждом экране), нижний абзац-сноску,
  `subtitle` подарка с карточки.
- **Свернуть:** оставить одну сетку «Вся коллекция» сразу, отсортированную по
  престижу. Лимитка/остаток — единственная метка, которая реально влияет на
  решение «брать сейчас».
- **Видно сразу:** сетка подарков 2 в ряд (mobile) с первого экрана. Hero — одна
  строка-заголовок.

### 1.4 Профиль (`components/v2/profile-v2.tsx`)

Самая длинная страница: Hero-обложка → 4 hero-стата → Витрина (до 6 карточек) →
Титул и статус (2 карточки) → Достижения (`AchievementsExperience`) → Подарки
(до 6) → Статистика (8 карточек) → История (до 20 событий) → ссылка назад.

- **Перегружено:**
  - Обложка `h-40`/`h-52` + аватар `h-28` — половина первого экрана уходит на
    декоративную шапку до того, как видно хоть один факт.
  - **Дублирование статов:** 4 «hero-стата» (MMR, место, баланс, достижения) и
    отдельная секция «Статистика» с 8 карточками, где **снова** баланс и
    достижения. Игрок видит баланс дважды, достижения дважды.
  - «Витрина» и «Подарки» и «История» во многом пересекаются по данным (всё из
    `activity`-ленты): легендарные моменты, подарки и таймлайн — три представления
    одного потока.
  - «Титул и статус» — две большие карточки ради одного титула и пары эмодзи-бейджей.
- **Удалить:** дубли статов (оставить один блок), отдельную карточку «Бейджи»
  если их нет; ссылку «← К жизни сообщества» (для этого есть bottom-nav).
- **Свернуть:** обложку до `h-24`; «Статистику» из 8 в 4–6 ключевых; «Историю»
  до 5–6 событий с кнопкой «показать ещё»; «Витрину» в горизонтальный скролл-ряд.
- **Видно сразу:** аватар + имя + титул + 3–4 ключевых факта (MMR, место, баланс)
  в компактной шапке без гигантской обложки.

### 1.5 Live (`app/live/page.tsx`)

12 секций подряд:
`PageHero → LiveNav → LiveCommunityStats → CommunityActivity → TopRich →
WeeklyTop → MessagesPanel → FamiliesTop → EconomyPanel → AchievementsCatalog →
TitlesLadder → DailyPanel → BotFeatures → CommandsExplorer → SiteFooter`.

Это не страница — это **вся вики проекта на одном бесконечном скролле.**

- **Перегружено:** всё. 12+ полноразмерных панелей. `LiveNav` (якорная
  навигация) сам по себе — признак того, что страница слишком длинная и её
  пришлось снабдить оглавлением.
- **Удалить с главного экрана Live:** `BotFeatures`, `CommandsExplorer`,
  `AchievementsCatalog`, `TitlesLadder` — это справочник, не «live». Они должны
  быть отдельными под-экранами/вкладками, а не лежать в одном скролле.
- **Свернуть:** разбить на табы внутри Live (Статистика · Топы · Экономика ·
  Справочник). Каждый таб — один экран, а не 1/12 бесконечной ленты.
- **Видно сразу:** живые цифры (`LiveCommunityStats`) + лента событий. Это и есть
  «live». Остальное — по табам.

## 2. Новый дизайн кейсов (компактная карточка)

**Цель:** на экране телефона одновременно 2–3 кейса.

### 2.1 Что показывает карточка (и только это)

Из `CaseView` (`lib/cases-ux.ts`) всё уже посчитано — новых данных не нужно:

| Элемент | Источник | Зачем |
|---|---|---|
| Название | `c.name` | идентификация |
| Цена | `costLabel(c)` | стоимость решения |
| Главный джекпот | `c.topReward.label` | «ради чего крутить» |
| Шанс Gift | `c.giftChance` | главный крючок |
| Шанс Premium | `c.premiumChance` | главный крючок |
| Кнопка «Открыть» | `CaseOpener` (idle) | действие |

**Всё остальное** (описание, полный список наград, ленты редкостей, «шанс редкой
награды», превью 4 наград) → в **модалку/экран деталей** по тапу на карточку.

### 2.2 Предлагаемый макет компактной карточки

```
┌─────────────────────────────────────┐
│ 📦  Название кейса          250 ешек │  ← иконка + имя + цена в одну строку
│ 💎 Главный приз: AirPods             │  ← один топ-джекпот, truncate
│ 🎁 Gift 4%   ⭐ Premium 1.2%         │  ← два чипа шансов, мелкие
│ [        Открыть  (250 ешек)       ] │  ← основная кнопка
└─────────────────────────────────────┘
```

- Высота цели: **~140–160px** вместо текущих ~520px+. Это даёт 2–3 карточки на
  экран телефона 700–800px рабочей высоты.
- Капсула-эмодзи: убрать `h-32` блок; вместо него эмодзи `📦` 28–32px слева в
  строке заголовка.
- Сетка: на мобиле `grid-cols-1` остаётся, но карточки низкие → влезает 3 шт.
  (Альтернатива — `grid-cols-2` с ещё более плотной карточкой; начать с 1 колонки
  и низкой высоты.)
- Рулетка (`case-opener.tsx`) остаётся как есть — она открывается **поверх**
  (в модалке/расширении карточки) при нажатии «Открыть». Сейчас она и так
  рендерится внутри карточки по фазам — нужно лишь поднять её в оверлей, чтобы
  спокойная сетка карточек не прыгала.

### 2.3 Экран деталей кейса (куда уходит «всё остальное»)

Открывается тапом по карточке (не по кнопке). Bottom-sheet или `/cases/[code]`:
- описание кейса;
- полный список наград с шансами (текущий `RewardRow`, но в полном списке);
- легенда редкостей;
- сноска про честность шансов (одна на экран, не на каждой карточке).

Так карточка остаётся витриной решения, а любопытные проваливаются в детали.

## 3. Аудит визуального шума

По каждому элементу: **«помогает принять решение» (✅) или «занимает место» (❌)**.

### Повторяющиеся тексты
- ❌ Сноски-абзацы внизу Кейсов и Подарков («Шансы рассчитаны из весов…»,
  «Цена указана в ешках…»). Повторяют то, что и так видно. → одна строка в
  деталях.
- ❌ Описания в `PageHero` на каждой странице («Трать ешки — лови Gifts…»). Игрок
  читает это один раз в жизни. → убрать или свести к подзаголовку из 4–5 слов.
- ❌ Профиль: баланс и достижения показаны дважды (hero-статы + секция
  «Статистика»). → один источник.
- ❌ Подарки: блок «Трофеи» = первые элементы «Всей коллекции». → убрать дубль.

### Лишние подписи
- ❌ `case-card`: заголовок «Лучшие награды», «и ещё N наград внутри», «Шанс
  редкой награды». → в детали.
- ❌ `gift-card`: `subtitle` (описание подарка), подпись «забрали N» мелким
  текстом. → оставить только цену + лимит/остаток.
- ❌ Профиль: «Текущий титул», «Бейджи» как заголовки внутри карточек —
  самоочевидно. → иконка + значение.
- ✅ Цена, шанс Gift/Premium, остаток лимитки — оставить, они влияют на решение.

### Лишние бейджи
- ❌ Пульсирующий «live»-бейдж в `PageHero` и `LiveStats` («Сейчас в ВОЗНЕ»,
  «Обновляется в реальном времени»). Анимация привлекает внимание к подписи, а не
  к контенту. → убрать или сделать статичной микрометкой.
- ❌ «Легенда редкостей» на Подарках — цвет рамки уже кодирует редкость.
- ✅ Метка «Лимитка» / «💎 джекпот» — помогает, оставить (но одна на карточку).
- ⚠️ Бейдж редкости (`RarityBadge`) на каждой карточке + цветная рамка + glow +
  цветная цена = редкость закодирована **четырьмя** способами. Оставить один-два
  (рамка + цена), убрать дублирующий текстовый бейдж на плотных карточках.

### Лишние блоки
- ❌ Главная: `About`, `FinalCta`, дубль масштаба (`Hero`/`LiveStats`/`TopMembers`),
  `StickyCta` (конфликт с bottom-nav).
- ❌ Live: `BotFeatures`, `CommandsExplorer`, `AchievementsCatalog`,
  `TitlesLadder` на главном скролле (→ в табы/справочник).
- ❌ Кейсы: блок «гид новичку» (4 плитки), `RarityStrip` на карточках.
- ❌ Кейсы desktop-`aside` («Лучшие открытия» / «Недавно открыли») на мобиле
  падает в самый низ длинной колонки — мало кто доскроллит. → короткий тикер
  сверху или убрать на мобиле.

## 4. Превратить сайт в приложение

Чтобы ощущалось как Telegram Mini App / мобильная игра / ежедневное приложение,
а не набор красивых страниц:

1. **Один герой на всё приложение, а не на каждой странице.** Убрать `PageHero`
   с `text-6xl` заголовками на Кейсах/Подарках/Live. Вместо него — тонкий
   заголовок-строка в шапке экрана (как title bar в приложении). Заголовок
   `text-6xl` уместен на лендинге, не в приложении.
2. **Первый экран = контент, а не объяснение.** На каждом экране сразу видно то,
   ради чего на него пришли (кейсы — кейсы, подарки — подарки), без «гидов» и
   абзацев сверху.
3. **Плотность вместо воздуха.** Текущие `py-12 sm:py-20`, `p-5`, `gap-4`,
   `rounded-3xl` — это лендинговые отступы. Для приложения: `py-3/4`, `p-3`,
   `gap-2`, `rounded-2xl`. Контент-плотность важнее «дыхания».
4. **Persistent header с балансом.** В приложении всегда видно «сколько у меня
   ешек». Сейчас баланс живёт только в профиле и в результате открытия. Вынести
   баланс в `SiteHeader` (постоянно сверху) — это ядро ежедневного приложения.
5. **Bottom-nav — единственная плавающая панель.** Убрать `StickyCta`, чтобы
   снизу не было двух конкурирующих CTA.
6. **Детали — в bottom-sheet, а не в новый длинный экран.** Открытие деталей
   кейса/подарка модалкой снизу = ощущение native-app, отсутствие «ушёл на другой
   лендинг».
7. **Мгновенная реакция.** Скелетоны вместо пустоты (в `LiveStats` уже есть —
   распространить на Кейсы/Подарки), тактильность кнопок (`active:scale` уже
   частично есть). Никаких «появляющихся при скролле» reveal-анимаций на контенте
   приложения — `framer-motion whileInView` уместен на лендинге, в приложении он
   создаёт ощущение медленной загрузки.
8. **Табы внутри тяжёлых разделов (Live).** Вместо 12 секций — 3–4 таба. Это
   паттерн приложения, а не статьи.
9. **Постоянство состояния.** Возврат на экран не должен прокручивать в начало
   длинной простыни — что естественно решается укорачиванием экранов.

Итог: после этих изменений переход между Главная/Кейсы/Подарки/Live ощущается как
переключение вкладок приложения, где каждая вкладка сразу показывает дело, а не
как пролистывание серии посадочных страниц.

## 5. Приоритетный план

Без новых механик. Только UX-плотность и иерархия.

### P0 — обязательно сейчас (страница «кейсы» и общий app-feel)
- [ ] **Компактная `CaseCard`** (раздел 2): только имя, цена, топ-джекпот, шанс
  Gift, шанс Premium, кнопка. Убрать описание, `RarityStrip`, список «Лучшие
  награды», «шанс редкой награды». Цель — 2–3 кейса на экран.
- [ ] **Детали кейса в bottom-sheet** (полный список наград + шансы + сноска).
- [ ] **Убрать блок «гид новичку»** и нижнюю сноску на Кейсах.
- [ ] **Сжать `PageHero`** на Кейсах/Подарках/Live до тонкой строки-заголовка
  (убрать абзацы-описания и `text-6xl`).
- [ ] **Убрать `StickyCta`** на мобиле (конфликт с bottom-nav).
- [ ] **Баланс в `SiteHeader`** (постоянно виден залогиненному).

### P1 — желательно
- [ ] **Главная → дашборд:** убрать `About`/`FinalCta`, свести
  `Platforms`+`BotEcosystem` в один компактный блок, `LiveStats` 6→3 метрики,
  поднять быстрые входы (Кейсы/Подарки/Live) на первый экран.

## Резюме

Корень проблемы — **лендинговая ДНК на каждом экране**: большой герой, абзацы-
объяснения, дублирующиеся «блоки доверия», декоративные элементы, не влияющие на
решение. Самый дорогой по эффекту шаг — **P0 по кейсам**: компактная карточка
(имя/цена/джекпот/Gift/Premium/кнопка) + детали в bottom-sheet даёт 2–3 кейса на
экран и сразу превращает главную игровую поверхность из лендинга в приложение.
Дальше тот же принцип («сначала контент, объяснения — в детали, плотность вместо
воздуха») тиражируется на Главную, Подарки, Профиль и Live.
