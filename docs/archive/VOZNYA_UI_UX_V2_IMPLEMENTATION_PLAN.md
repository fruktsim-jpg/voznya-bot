# VOZNYA UI/UX V2 — Implementation Masterplan

Спутник к `VOZNYA_UI_UX_V2.md` (главный UX-документ). Здесь — **как** превратить
текущий сайт в V2 с минимумом переделок и максимумом переиспользования кода.
Только аудит и план. **Код не трогаем.** Backend/миграции/экономика/кейс-логика/
админка/аналитика — вне зоны (всё ниже — слой представления поверх данных,
которые система уже отдаёт).

Опорные факты по коду (проверено):
- Сайт строго **read-only** над Postgres бота; бот — единственный писатель; все
  лоадеры безопасно деградируют (`[]`/`null`/0) на немигрированной БД.
- Tailwind v4, токены inline в `globals.css` (`--background:#09090b`,
  `--primary:#8b5cf6`, `--accent:#a855f7`, `.glass`, `.text-gradient`), Geist.
- `components/` ~39 файлов в папках `voznya/`, `live/`, `profile/`, `auth/`,
  `admin/`, `ui/` (в `ui/` только `button` + 1 примитив).
- Главные данные: `lib/queries.ts` (community/economy/top-rich/weekly/families/
  achievements/messages/daily/MMR), `lib/cases.ts`, `lib/gifts.ts`,
  `lib/economy-analytics.ts`, `lib/mmr.ts` (RANKS).
- Feed-able события уже пишутся ботом: `case_openings`, `gift_transactions`,
  `transactions` (casino/purchase), `user_achievements`, `marriages`,
  `reputation_entry`, `mmr_entry`, `stars_ledger` — все с `created_at`.

## Часть 3. Новый дизайн-слой

### Система редкостей (единая: кейсы / подарки / достижения / бейджи)
Источник тиров: `inventory_items.rarity`, `case_rewards.is_jackpot/max_global_supply`,
`gift_catalog.star_cost` (тир по цене), MMR `RANKS` из `lib/mmr.ts`. Единая шкала:

| Тир | Цвет | Glow | Где |
|---|---|---|---|
| Common | серый `#9ca3af` | нет | частые дропы/предметы |
| Uncommon | зелёный `#22c55e` | мягкий | дешёвые подарки |
| Rare | синий `#3b82f6` | заметный | средние |
| Epic | фиолетовый `#8b5cf6` (=primary) | сильный | дорогие/редкие |
| Legendary | золотой `#f59e0b` | сильный + частицы | топ-дропы/премиум |
| Mythic / Telegram Gift | градиент violet→gold + свечение | максимум | реальные Telegram Gifts, джекпоты |

Реализация: `ui/rarity` — `rarityToken(tier) → {border, glow, label, textClass}`.
Применяется к рамке любой `Card`-награды. Не вводит новых цветов вне бренда —
расширяет существующую палитру, не превращает в Web3/казино-клон.

### Типы карточек (единый `Card`, варианты)
- **event** — событие ленты: аватар + действие + время + (опц.) рамка редкости.
- **player** — игрок: аватар + ник + титул + ключевая метрика (баланс/MMR).
- **case** — кейс: визуал + цена открытия + тизер лучшего дропа.
- **gift** — подарок: визуал + цена в ешках + тир/редкость + остаток/лимит.
- **achievement** — ачивка: иконка + название + прогресс/статус (получено/нет).
- **rank/leaderboard** — строка рейтинга: позиция + игрок + значение (табличный вид на desktop).

Различия — только в наполнении и рамке; общий glass-контейнер, радиус, hover-lift.

### Цветовая система
- База без изменений: dark `#09090b`, primary `#8b5cf6`, accent `#a855f7`, `.glass`.
- Добавить **семантику как токены**: success `#22c55e`, warn `#f59e0b`, danger `#ef4444`, info `#3b82f6` — для статусов доставки подарков, экономики, состояний.
- Редкости (см. выше) как отдельная группа токенов.
- Принцип: не ломать идентичность Возни; редкости/семантика — аддитивны.

## Часть 4. Лента событий (ядро V2)

### Зачем
Главный рычаг ощущения «тут постоянно движуха». Чистый UI поверх данных, которые
бот **уже пишет** — никаких изменений в боте/экономике.

### Источники (уже существуют, все с `created_at`)
| Событие | Таблица | Что показать |
|---|---|---|
| Открыл кейс | `case_openings` | «{игрок} открыл {кейс} → {награда}» + редкость (join `inventory_items.rarity`); джекпот по `case_rewards.is_jackpot` |
| Подарок | `gift_transactions` (`status='completed'`) | «{отправитель} подарил {подарок} {получателю}»; `tg_gift` — особый «золотой» тип |
| Крупный выигрыш | `transactions` (`reason='casino'`, по `meta.payout`) | «{игрок} выиграл {сумма} в казино» (порог отсечки) |
| Достижение | `user_achievements` | «{игрок} получил достижение {название}» |
| Новая семья/пара | `marriages` | «{A} и {B} создали семью» |
| Репутация (опц.) | `reputation_entry` | «{игрок} получил репутацию» |
| Повышение ранга (опц.) | `mmr_entry` | «{игрок} поднялся до {ранг}» |

Реализация: `lib/feed.ts` — `getCommunityFeed({limit, types})` = `UNION ALL`
нормализованных выборок (тип, actor, target, value, rarity, created_at) ⟶ сортировка
по времени. Деградирует как остальные лоадеры. Опрос — periodic refetch/ISR (без
сокетов на старте).

### Структура события (карточка `event`)
аватар актора · текст действия · бейдж типа · (опц.) рамка/иконка редкости ·
относительное время («2 мин назад»). Клик → профиль актора / страница кейса.

### Расположение
- **Главная:** компактная лента-тизер (5–8 последних, «Смотреть всё» → `/live`).
- **`/live`:** полноценная лента сверху, бесконечная/постранично.

### Mobile
Вертикальный список, карточки в ширину экрана, новые элементы въезжают сверху
(fade/slide), pull-to-refresh. Фильтры — горизонтальные чипы (Всё/Кейсы/Подарки/
Казино/Достижения/Семьи).

### Desktop
Лента в колонке (часто sticky сбоку от рейтингов), фильтры — таб-строка.
Группировка по времени («Сегодня», «Вчера»).

### Фильтры
По типу события (чипы). Состояния: загрузка — скелетоны карточек; пусто —
«Пока тихо, но скоро здесь будет жарко».

## Часть 5. Новый профиль (Steam-профиль Возни)

Цель: личный паспорт, который хочется прокачивать. Данные уже отдаёт
`getPlayerProfile`/`lib/queries.ts` (баланс, total earned, ранг/MMR, репутация,
достижения) — меняем только композицию, без бэкенда.

### Hero-блок (видно сразу)
- крупный аватар + ник + **титул** (статусный бейдж);
- ранг/MMR + прогресс-бар «до следующего ранга» (RANKS из `lib/mmr.ts`);
- ключевые метрики строкой: баланс · total earned · репутация · позиция в топе;
- ряд топ-**бейджей** (лучшие достижения) как витрина;
- owner: кнопки «Открыть в Telegram», «Поделиться»; admin: служебный блок (как сейчас).

### Вкладки (`ui/tabs`)
1. **Обзор** — текущий `PlayerCard`-контент (сводка + последняя активность).
2. **Достижения** — все ачивки, полученные выделены, остальные с прогрессом/замком.
3. **Коллекция** — инвентарь/косметика/подарки карточками с редкостью; пустые слоты = стимул.
4. **Статистика** — экономика игрока, казино/дуэли/ферма, графики (из существующих данных).
5. **История** — персональная лента (его `case_openings`/`gift_transactions`/`transactions`).

### Мотивация прокачки
Прогресс-бары (анимированные), пустые слоты коллекции, редкие бейджи визуально
«дороже» (рамка редкости), сравнение с топом.

### Реализация
`profile/profile-hero` + `profile/tabs/{overview,achievements,collection,stats,history}`.
`PlayerCard` рефакторится: его текущая разметка → вкладка «Обзор». Роут можно
оставить `/profile/[id]` (или ввести `/u/[id]` с редиректом старого — косметика).

## Часть 6. Cases Experience (важнейший игровой раздел)

### Каталог `/cases` (или `/play/cases`)
Сетка `case-card`: визуал, цена открытия, тизер лучшего дропа, бейдж «новый/
лимитированный». Данные — `lib/cases.ts`.

### Страница кейса `/cases/[code]`
- Содержимое: все возможные награды карточками с **редкостью** (join `inventory_items`).
- **Шансы**: проценты по тирам/наградам (из `case_rewards` весов) — прозрачно.
- Цена открытия, кнопка «Открыть в Telegram» (дип-линк; действие — в боте).
- «Последние открытия этого кейса» — мини-лента из `case_openings` по `case_item_code`.

### История открытий
Общая — на `/live`; персональная — в профиле (вкладка «История»).

### Отображение редкостей/шансов
Единый `ui/rarity`; шансы — `cases/odds-table` (тир → %).

### UX будущей анимации открытия (Mini App, до этого — дип-линк)
**Результат определяется сервером заранее; анимация только визуализирует уже
полученный результат.**
- Прокрутка горизонтальной ленты предметов, ease-out, маркер по центру, ~4–6 c.
- Цвет дорожки/вспышки по редкости; чем реже — ярче и дольше торможение.
- На остановке: вспышка/частицы цвета тира, зум на приз, конфетти для топа.
- Звук: тики прокрутки (уск.→замедл.), удар на стопе, отдельный «триумф».
- Haptics (Telegram Mini App): тики при прокрутке, сильный импульс на результат.
- Праздничные состояния: полноэкранный момент для Legendary/Telegram Gift
  («🎁 тебе выпал реальный подарок» + статус выдачи из `gift_transactions`).
Компоненты-задел: `cases/opening-reel`, `cases/opening-result` (живут в Mini App слое).

## Часть 7. Gifts Experience (с учётом Gifts Shop V1)

Бэкенд готов: каталог `gift_catalog`, покупка за ешки (атомарно), выдача `sendGift`,
`stars_ledger`, статусы `gift_transactions`. UI:

- **Магазин** `gifts/gift-card`: визуал, имя, **цена в ешках**, тир/редкость (из `star_cost`), остаток/лимит, кнопка «Купить в Telegram». Премиум/статусные — крупнее, со свечением.
- **Редкости/тиры**: единый `ui/rarity`; «Лимит» + счётчик остатка; распродано → приглушённая карточка «Раскуплено».
- **Инвентарь** (профиль → Коллекция): полученные подарки с датой/редкостью.
- **История** (профиль → История или отдельный таб): покупки/выдачи со статусами (оплачено/отправлено/возврат) — из `gift_transactions`/`purchase_history`.
- **Стоимость**: показываем в ешках; «реальность» — визуалом (настоящий Telegram Gift), без раскрытия себестоимости в Stars игроку.

## Часть 8. Casino Experience (без дешёвого гемблинга)

Сейчас механика — в боте; данные — `transactions reason='casino'` (`meta.bet/payout/
outcome`), плюс `loadCasinoOverview/Extremes`. На сайте делаем сдержанную витрину:

- **Хаб** `/casino` (или `/play/casino`): режимы карточками + краткие правила, кнопка «Играть в Telegram». Тон премиальный, в общем dark/violet, без мигающих джекпотов.
- **Моя статистика**: сыграно/выиграно/проиграно, RTP игрока, серии (новый read-loader `lib/casino.ts` поверх существующих транзакций).
- **История игр**: последние спины игрока.
- **Крупные выигрыши сообщества**: `loadCasinoExtremes` — социальное доказательство (и в ленту событий).
- **Лидерборды**: топ по выигрышам/активности.
- Принцип: статистика и соревнование, не «занеси ещё».

## Часть 9. Mobile First (как будущий Mini App)

- **Нижняя навигация** `nav/bottom-nav` (fixed): `Дом · Live · Играть · Профиль`, иконки lucide + подписи, активный таб = `--primary`. Скрыта на desktop. Главный носитель app-ощущения.
- **Паттерны:** скелетоны, оптимистичные переходы, sticky «Открыть в Telegram», bottom-sheet для деталей вместо модалок.
- **Размеры:** тач-таргеты ≥44px, крупные карточки кейсов/подарков (сетка 2 в ряд), читаемые шрифты.
- **Жесты:** свайп между вкладками профиля/рейтингов, pull-to-refresh на `/live` и ленте, тап-scale на карточках.
- **Сценарии:** «зашёл с телефона → нижний таб Live → лента движухи → тап на игрока → профиль → Открыть в Telegram». Всё в 2–3 тапа.

## Desktop (кратко, без растягивания мобильного)
- `/live`: рейтинги 2–3 колонки + лента sticky сбоку.
- Профиль: hero/бейджи слева sticky, контент вкладок справа.
- Каталоги: 3–4 карточки в ряд; страница кейса — содержимое + шансы рядом.
- Рейтинги — полноценные таблицы с сортировкой.

## Часть 10. Mini App Roadmap (что куда)

- **Остаётся на сайте:** портал-главная, Live Center, рейтинги, публичный профиль (SEO), каталоги кейсов/подарков как витрины, привязка аккаунта, казино-витрина/статистика.
- **Переезжает/появляется в Mini App:** интерактивные действия внутри Telegram — открытие кейса с анимацией §6 + haptics, покупка подарка не выходя из TG, быстрый профиль/баланс, казино-экраны.
- **Общее (одна дизайн-система):** `ui/card`, `ui/rarity`, токены, event-feed, profile-hero — пишем переиспользуемо, чтобы Mini App забрал их без переделки.
- **Только в чате:** живое общение, мемы, сходки, социальные команды.
- **Бот:** единственный писатель БД и оркестратор (из чата и из Mini App).

До Mini App все игровые кнопки на сайте — дип-линки «Открыть в Telegram».

## Часть 12. Итоговый Roadmap (по пользе, не по красоте)

**Шаг 1 — Дизайн-система + нижняя навигация.**
Почему первым: дёшево, низкий риск, разблокирует все остальные экраны и сразу даёт
app-ощущение. Создаём `ui/card`, `ui/rarity`, `ui/tabs`, `ui/skeleton`,
`ui/empty-state`, `ui/badge`, `nav/bottom-nav`. Каркас `/play`.

**Шаг 2 — Лента событий.**
Почему вторым: главный рычаг «движухи», максимальное влияние на вовлечение/удержание,
данные уже есть (`lib/feed.ts` поверх существующих таблиц). Тизер на главной + полная на `/live`.

**Шаг 3 — Переосмысление главной + иерархия `/live`.**
Почему: превращает «сайт бота» в «портал сообщества» (рост/конверсия в чат), и
после ленты это естественный следующий шаг. Реордер блоков, понижение
`Features/Bonuses`, справку `/live` — вниз.

**Шаг 4 — Cases Experience (каталог → страница кейса + шансы + история).**
Почему: ядро экономики, высокая вовлечённость, действие пока дип-линк (низкий риск).

**Шаг 5 — Профиль-паспорт.**
Почему: главный драйвер долгого удержания («хочу прокачать»), но дороже (рефактор
`PlayerCard` во вкладки) — после того как готова дизайн-система и редкости.

**Шаг 6 — Gifts + Casino витрины.**
Почему: усиливают игровую ценность и соцдоказательство; опираются на уже готовые
данные (Gifts Shop V1, casino-транзакции).

**Шаг 7 — Mini App: интерактив + анимация открытия.**
Почему последним: максимальная эмоция, но самый дорогой и рискованный слой;
переиспользует дизайн-систему и компоненты из шагов 1–6 без переделок.

Принцип всего плана: сначала дешёвый разблокирующий фундамент и «живость», затем
игровая ценность и профиль (удержание), и в конце Mini App. Backend/экономику не
трогаем — весь план это слой представления поверх данных, которые система уже
отдаёт, с максимальным переиспользованием существующих компонентов и лоадеров.
