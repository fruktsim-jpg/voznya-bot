# Release 2.2 — Site-first: карта команд и кнопок бота

Принцип: если действие сложнее просмотра данных — игрок идёт на сайт. Telegram
остаётся для уведомлений, социалки и быстрых команд. Сайт — единый центр опыта
(кейсы, магазин, инвентарь, Premium, подарки, коллекции, статистика).

URL-кнопки строятся хелпером `app.core.keyboards.open_on_site(label, url)` на
базе `settings.website_url` (по умолчанию `https://voznya.nl`).

## Перенесено на сайт (бот показывает карточку + кнопку)

| Команда / кнопка | Было | Стало | Ссылка |
|------------------|------|-------|--------|
| `/кейсы`, `/cases` | список + открытие в чате | карточка «Кейсы на сайте» + кнопка | `/cases` |
| `/открыть`, `/open` | вскрытие кейса в чате | карточка + кнопка на сайт | `/cases` |
| `/кейс <код>` | дроп-лист + кнопка «Открыть» (в чате) | дроп-лист (просмотр ок) + кнопка «Открыть кейсы» на сайт | `/cases` |
| `/магазин`, `/shop`, `/подарки`, `/gifts` | витрина + покупка в чате | карточка «Магазин Возни» + кнопка | `/gifts` |

Старая механика открытия (`cb_case_open`, `_open_with_animation`) и витрина
(`_shop_keyboard`, `cb_gift_buy`) в коде сохранены — их использует внутренний
конвейер и обратная совместимость, но новые сообщения их больше не вешают.

## Остаётся в Telegram (просмотр данных) + кнопка на сайт

| Команда | Роль в боте | Кнопка на сайт |
|---------|-------------|----------------|
| `/инвентарь`, `/инв`, `/рюкзак` | быстрый просмотр стека + pending-подарков | «🎒 Открыть полный инвентарь» → `/inventory` (только свой) |
| `/профиль`, `/кто` | карточка профиля | «🌐 Открыть профиль на сайте» → `/profile/<id>` (свой и чужой) |
| `/топ` | рейтинг богачей + пагинация | (пагинация в чате; CTA на сайт можно добавить) → `/live` |
| `/топнеделя` | топ за неделю | «📊 Полная статистика на сайте» → `/live` |
| `/семьи` | топ браков | «📊 Полная статистика на сайте» → `/live` |
| `/моиподарки` | статус своих покупок | — (чистый просмотр) |

## Остаётся только в Telegram (соц./быстрые/уведомления)

Не дублируется на сайте — это родная для чата механика:

- Экономика чата: `ферма`, `баланс`, `казино <ставка>`, `клад`.
- Социалка: `бой @ник`, `жениться`/`да`/`брак`/`расстаться`, `пидор`, `пара`,
  `осеменить`, `реп`, `кто ты`.
- Прогресс: `ачивки`, `ммр`, `топммр`, `топреп`.
- Уведомления: выпадения, доставки, достижения, события клада/дуэли.

## Админ — остаётся в Telegram (оперативный инструмент)

Выдача подарков и модерация — быстрее из чата, дублировать на сайт не нужно
(на сайте есть отдельная админка deliveries):

- `/gifts_pending` (+ инлайн-кнопки 🔁 Повторить / ✅ Выдать / ↩️ Возврат),
  `/gifts_done`, `/gifts_retry`, `/gifts_refund`.
- `/gifts_available`, `/gifts_setid`.
- `/выдать`, `/забрать`, `/инфо`, `/инфо`-команды.

## Итог изменений в коде

- `app/core/keyboards.py` — хелпер `open_on_site`.
- `app/features/cases/handlers.py` — `/кейсы`, `/открыть` → карточка-редирект;
  `/кейс` сохраняет дроп-лист, но кнопка ведёт на сайт.
- `app/features/gifts/handlers.py` — `/магазин` и алиасы → карточка-редирект.
- `app/features/inventory/handlers.py` — кнопка на полный инвентарь (свой).
- `app/features/ratings/handlers.py` — CTA «Полная статистика» под `/топнеделя`
  и `/семьи`.
- `app/features/profile/handlers.py` — кнопка профиля на сайте (уже была).
- Тексты: `texts.CASES_SITE_*`, `texts.TOP_STATS_SITE_BTN`,
  `gifts.SHOP_SITE_*`, `inventory.INV_SITE_BTN`.
