# Пополнение Stars для бота — аудит + практический сценарий первой выдачи

Сверено с Bot API (`https://core.telegram.org/bots/api`, разобрана 2026-06-07).
Где факт подтверждён докой — помечено ✅; где это политика Telegram вне Bot API
(меняется, проверять в приложении) — помечено ⚠️ «проверить в Telegram».

---

## 1. Как бот может получить Stars

✅ **Главный и фактически единственный способ через Bot API — платные инвойсы в
Stars.** Бот выставляет счёт в валюте `XTR` (Telegram Stars), пользователь его
оплачивает — Stars зачисляются на баланс бота. Методы:
- `createInvoiceLink` (#createinvoicelink) — `currency='XTR'`, `provider_token=''`
  (пустая строка для оплат в Stars) → возвращает ссылку на счёт.
- `sendInvoice` (#sendinvoice) — то же, но отправляет счёт сообщением в чат.
- Подтверждение оплаты: апдейты `pre_checkout_query` (ответить
  `answerPreCheckoutQuery`) и затем `message.successful_payment` с
  `telegram_payment_charge_id`.

⚠️ Stars не «дарятся» боту администратором напрямую — **нет API «положить Stars
на баланс бота»**. Они появляются только как оплата счёта. Поэтому «пополнение
владельцем» = владелец сам оплачивает счёт, выставленный ботом (см. §5).

## 2. Где посмотреть Star Balance бота

✅ `getMyStarBalance` (#getmystarbalance) → `StarAmount{amount}`. В нашем коде
обёрнут в `app/services/telegram_gifts.get_star_balance`.
✅ `getStarTransactions` (#getstartransactions) → история Star-операций бота
(приходы от оплат, расходы на `sendGift`). В коде пока НЕ вызывается (задел под
reconcile).

## 3. Можно ли вручную пополнить баланс бота владельцем

⚠️ Прямого «депозита» нет. Но владелец может **оплатить счёт собственного бота**
своими Stars — это и есть ручное пополнение:
1. Владелец покупает Stars в приложении Telegram (Настройки → ... → Stars, оплата
   через Apple/Google/в некоторых регионах иначе).
2. Бот выставляет счёт в `XTR` (через `createInvoiceLink`/`sendInvoice`).
3. Владелец оплачивает этот счёт — Stars уходят с его личного баланса и
   зачисляются боту.

То есть «пополнить бота» технически = «оплатить боту счёт». Других путей через
Bot API нет.

## 4. Как работает экономика Stars для ботов (кратко)

- ⚠️ Пользователи **покупают** Stars у Telegram (через сторы). Часть стоимости
  забирает площадка (Apple/Google) — это комиссия на стороне покупки Stars, не на
  стороне бота.
- ✅ Бот **получает** ровно номинал Stars, уплаченный за его цифровой товар/услугу
  (через XTR-инвойс).
- ✅ Бот **тратит** Stars: на `sendGift` (наш кейс), платные медиа и пр. — списание
  с баланса бота.
- ⚠️ **Вывод** заработанных Stars (конвертация в TON/деньги через Fragment) —
  доступен с задержкой (часто упоминается ~21 день удержания) и со своими лимитами.
  Это про вывод, НЕ про трату: тратить на `sendGift` можно сразу, как Stars на
  балансе. Точные сроки/лимиты — проверять в Telegram, они вне Bot API и меняются.

## 5. Самый простой способ получить первые Stars для теста

Практически: **оплатить боту минимальный XTR-счёт самим собой.**
1. Купить в приложении Telegram небольшое количество Stars (с запасом под
   `star_cost` тестового подарка — напр. наш `gift_heart` стоит 15★).
2. Бот должен уметь выставить счёт в Stars (`createInvoiceLink`,
   `currency='XTR'`, `provider_token=''`) и принять оплату (`pre_checkout_query`
   + `successful_payment`).
3. Открыть ссылку счёта, оплатить своими Stars → баланс бота вырастет.
4. Проверить: `getMyStarBalance` (или наш экран/лог) показывает баланс ≥ star_cost.

⚠️ Важная оговорка о реальности: бот **не может** получить Stars без шага «кто-то
оплатил его счёт». Поэтому без инвойс-флоу пополнить баланс нечем.

## 6. Ограничения и комиссии

- ✅ `createInvoiceLink`/`sendInvoice` для Stars: `provider_token` пустой,
  `currency='XTR'`.
- ⚠️ Комиссия площадки — на покупке Stars пользователем (Apple/Google), не у бота.
- ⚠️ Вывод Stars — с задержкой и лимитами (Fragment); трата на `sendGift` —
  без задержки.
- ✅ `sendGift` спишет Stars с баланса бота; не хватает баланса → ошибка (наш
  адаптер заранее проверяет `getMyStarBalance` и не пытается слать впустую).
- ⚠️ Минимальные/максимальные суммы Star-инвойса и доступность покупки Stars в
  конкретном регионе — проверять в Telegram.

## 7. Что из этого УЖЕ поддерживает наш код

| Возможность | В коде сейчас |
|---|---|
| Проверка баланса бота (`getMyStarBalance`) | ✅ `telegram_gifts.get_star_balance` (с защитой от старой версии aiogram) |
| Трата Stars на подарок (`sendGift`) | ✅ `telegram_gifts.send_gift` (за флагом `GIFTS_DELIVERY_ENABLED`) |
| История Stars (`getStarTransactions`) | ⚠️ не вызывается (задел) |
| **Получение Stars (XTR-инвойс)** | ❌ **НЕ реализовано** — нет `createInvoiceLink`/`sendInvoice`, нет хендлеров `pre_checkout_query`/`successful_payment` |
| Донат Stars→ешки, `stars_ledger` | ❌ отдельный будущий этап |

**Вывод по коду:** тратить Stars бот умеет, а **получать — пока нет.** Чтобы
сделать первую реальную выдачу Gift, баланс бота должен быть пополнен, а
пополнение возможно только через оплаченный XTR-инвойс, которого в коде ещё нет.

## 8. Практический сценарий «получить первые Stars и выдать первый Gift сегодня»

Честный путь с учётом текущего состояния кода:

**Что нужно один раз сделать (это новый код — по твоему правилу «новых систем не
добавлять» НЕ делаю сейчас, фиксирую как следующий шаг):**
- Минимальный «топ-ап» флоу для владельца: команда у бота, которая шлёт
  `createInvoiceLink(currency='XTR', provider_token='', prices=[LabeledPrice(amount=N)])`,
  + хендлеры `pre_checkout_query` (ответить True) и `successful_payment` (залогировать
  `telegram_payment_charge_id`). Это же — фундамент будущего «донат Stars→ешки».

**Дальше (без нового кода):**
1. Купить Stars в приложении Telegram (с запасом ≥ `star_cost` подарка).
2. Оплатить счёт, который выставит бот (из флоу выше) → Stars на балансе бота.
3. В каталоге проставить **реальный** `telegram_gift_id` (из `getAvailableGifts`)
   для тестовой позиции — сид 0020 кладёт `telegram_gift_id=NULL`, с NULL выдача
   не пройдёт.
4. `GIFTS_DELIVERY_ENABLED=true`, перезапустить бота.
5. `/подарки` → купить → ожидать `status='completed'`, у получателя появится
   подарок; в `/admin/economy/gifts` вырастут «истрачено Stars» и «выдано».

**Альтернатива без своего инвойс-флоу:** проверить в приложении Telegram, есть ли
у конкретного бота через @BotFather/настройки опция пополнения баланса Stars
напрямую (⚠️ это вне Bot API и зависит от текущей политики Telegram — нужно
смотреть в приложении). Если такая опция есть — пополнить ею и сразу идти к шагу 3.

## 9. Рекомендация следующего шага

Так как «получать Stars» сейчас в коде нет, минимальный честный следующий шаг —
**топ-ап инвойс для владельца** (≈ один роутер: `createInvoiceLink` +

- одновременно становится фундаментом будущего «донат Stars→ешки» и `stars_ledger`
