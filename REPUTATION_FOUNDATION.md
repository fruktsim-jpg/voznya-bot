# REPUTATION SYSTEM V1

Социальный рейтинг сообщества «Возни». Игроки благодарят/осуждают друг друга
обычными словами в ответ (reply) на сообщение. Репутация полностью изолирована
от ешек, XP, счётчика сообщений, магазина, инвентаря, Combot и OIDC.

## Как это работает

Репутация выдаётся ТОЛЬКО ответом (reply) на чужое сообщение. Игрок отвечает
фразой-алиасом — автору исходного сообщения прилетает +1 или −1.

- Положительные фразы (`+реп`, `спасибо`, `респект`, `база`, ...) → +1
- Отрицательные фразы (`-реп`, `кринж`, `минус`, `пнх`, ...) → −1

Совпадение точное: весь текст ответа целиком должен быть одной фразой-алиасом
(с точностью до регистра и лишних пробелов). Длинные сообщения, где алиас —
лишь часть текста, не срабатывают, чтобы не плодить случайных оценок.

## Команды

- `реп` / `репутация` (`rep`) — карточка репутации. Без аргументов своя; в
  ответ на сообщение — репутация автора. Показывает итог, плюсы, минусы.
- `топреп` (`toprep`) — топ игроков по итоговой репутации.

## Ограничения

- нельзя оценивать самого себя;
- нельзя оценивать ботов (`from_user.is_bot`);
- нельзя оценивать удалённых/неизвестных боту (нет строки в `users`);
- одной командой — одно изменение (нет «накрутки» за один reply).

## Антиспам

Одну и ту же пару «оценивающий → оценённый» можно менять не чаще
1 раза в `REPUTATION_COOLDOWN_HOURS` (по умолчанию 12 ч). Другим людям тот же
игрок может ставить репутацию без ограничения. Проверка — по времени последней
записи этой пары в журнале (индекс `ix_reputation_pair`), отдельной таблицы
кулдаунов нет.

## Хранение

Источник правды — таблица `reputation_entries` (журнал). Одна строка = одно
изменение. Текущая репутация и топы — агрегаты `SUM(value)`, поэтому значение
всегда можно пересчитать из истории.

### Схема `reputation_entries`

| Колонка          | Тип            | Назначение                                  |
|------------------|----------------|---------------------------------------------|
| `id`             | BigInteger PK  | автоинкремент                               |
| `target_user_id` | BigInteger     | кого оценили                                |
| `giver_user_id`  | BigInteger     | кто оценил                                  |
| `value`          | SmallInteger   | +1 / −1                                     |
| `reason`         | String(64)     | фраза-алиас, вызвавшая изменение            |
| `created_at`     | DateTime(tz)   | когда (server default `now()`)              |

Ограничения и индексы:

- `ck_reputation_value` — `value IN (-1, 1)`;
- `ck_reputation_not_self` — `giver_user_id <> target_user_id`;
- `ix_reputation_target` — агрегаты/топы по получателю;
- `ix_reputation_pair` (`giver_user_id, target_user_id, created_at`) — антиспам.

Связь с `users` — логическая (по `user_id`), без FK: журнал самодостаточен и
переживает любые чистки пользователей. История (`get_history`) и пересчёт
(`get_summary`) читаются из этого же журнала.

## Файлы

| Файл                                         | Роль                                   |
|----------------------------------------------|----------------------------------------|
| `app/models/reputation_entry.py`             | модель `ReputationEntry`               |
| `app/settings/reputation.py`                 | алиасы, кулдаун, лимит топа, тексты     |
| `app/repositories/reputation.py`             | запросы: summary, антиспам, add, top, history |
| `app/features/reputation/service.py`         | `classify()` + `apply_reputation()`     |
| `app/features/reputation/handlers.py`        | reply-фильтр + команды                 |
| `migrations/versions/0013_reputation_foundation.py` | миграция таблицы                |

Регистрация: модель — в `app/models/__init__.py`; роутер — в
`app/features/__init__.py` (ДО командных роутеров, чтобы reply-алиасы вроде
«топ»/«+» уходили в репутацию, а обычные не-reply команды — дальше по цепочке).

## Миграция

```
docker compose exec bot alembic upgrade head     # создаст reputation_entries
docker compose exec bot alembic current          # → 0013_reputation_foundation
```

Цепочка ревизий: `0012_combot_import_foundation` → `0013_reputation_foundation`.
Откат: `alembic downgrade 0012_combot_import_foundation` (дропнет таблицу).

## Что НЕ затронуто

users, баланс/`transactions`, XP, `messages_count`, `cooldowns`,
`inventory`/`inventory_history`, `shop_*`, `gift_transactions`, `combot_*`,
OIDC, `account_links`, `admin_roles`, `audit_log`. Существующие команды не
изменены.

## Точки расширения (на будущее, без реализации сейчас)

- XP за репутацию — отдельный этап; сейчас связи нет.
- Mini App / сайт: читать `reputation_entries` напрямую (агрегаты `SUM(value)`),
  репозиторий уже даёт summary/top/history.
- Тонкая настройка: алиасы и `REPUTATION_COOLDOWN_HOURS`/`TOP_REPUTATION_LIMIT`
  правятся в `app/settings/reputation.py` без миграций.
- Возможные индексы при росте: частичные/`BRIN` по `created_at`, материализо-
  ванный агрегат репутации на пользователя, если `SUM` станет дорогим.
